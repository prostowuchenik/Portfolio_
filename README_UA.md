# Віртуальне портфоліо (шаблон) — Variant B

Цей шаблон зроблений так, щоб студент **не писав код**. Студент робить лише 3 дії:

1) Замінює `assets/room.glb` на свою кімнату (експорт з Blender у `.glb`).
2) Додає зображення робіт у папку `works/` у форматі `WORK_01.jpg`, `WORK_02.png` … (**UPPERCASE**).
3) Заповнює `config/config.json` (**UPPERCASE ключі**) — список робіт, слоти HOOK/FRM, матеріали, світло.

---

## 0) Що потрібно встановити (для локального запуску)

### Варіант A (найпростіший): VS Code + Live Server
1. Встановіть **Visual Studio Code**: https://code.visualstudio.com/
2. Запустіть VS Code → відкрийте папку `web-portfolio-template`.
3. Встановіть розширення **Live Server** (Ritwick Dey):
   - Extensions (іконка кубиків зліва) → пошук `Live Server` → Install.
4. Відкрийте `index.html` → правою кнопкою → **Open with Live Server**.
5. У браузері відкриється адреса типу `http://127.0.0.1:5500/`.

> Важливо: **не відкривати** `index.html` через `file://...` — тоді `config.json` та текстури не завантажаться через політики безпеки браузера.

### Варіант B (без VS Code): Python локальний сервер
Якщо встановлено Python 3:
- у папці проєкту виконайте:
  - Windows: `py -m http.server 8000`
  - macOS/Linux: `python3 -m http.server 8000`
- відкрийте у браузері: `http://localhost:8000/`

## 2) Як працюють “слоти” HOOK/FRM

У `room.glb` мають існувати:
- `FRM_01 … FRM_N` — 3D-рами (mesh)
- `HOOK_01 … HOOK_N` — “якорі” (Empty), які задають **центр** роботи та **орієнтацію** (куди дивиться площина)

У `config.json` у масиві `SLOTS` кожен слот описує:
- `HOOK_NAME` (наприклад, `HOOK_01`)
- `FRAME_NAME` (наприклад, `FRM_01`) — для узгодження з паспортом (в цьому шаблоні рамки не чіпаємо, але назва потрібна)
- `INNER_W`, `INNER_H` — “вікно” рами в метрах (куди вписати роботу)
- `IMAGE_OFFSET` — мікрозсув вперед (щоб картинка не “миготіла” з геометрією)

---

## 3) Як код “вписує” зображення у раму (Режим 1 — CONTAIN)

Шаблон використовує **CONTAIN**: зображення **не обрізається**, а масштабується так, щоб повністю вміститися у `INNER_W × INNER_H` зі збереженням пропорцій.

---

## 4) Як додати/змінити роботи

1. Додайте файли у `works/` з іменами `WORK_01.jpg`, `WORK_02.jpg` … (UPPERCASE).
2. Відкрийте `config/config.json`:
   - Додайте елемент у масив `WORKS` з полями:
     - `FILE` (напр. `WORK_11.jpg`)
     - `TITLE`, `AUTHOR`, `YEAR`, `TECHNIQUE`, `DESCRIPTION`
     - `SLOT_ID` (напр. `SLOT_11`) або пропустіть `SLOT_ID`, тоді система призначить слот по черзі.
3. Якщо робіт стало більше — додайте нові слоти у `SLOTS` (`SLOT_11`, `HOOK_11`, `FRM_11` …).

---

## 5) Як міняти колір/текстуру стін/підлоги/стелі без Blender

У `config.json` є `MATERIAL_OVERRIDES`. Приклад:
```json
{
  "MATERIAL_NAME": "MAT_WALL_01",
  "COLOR": "#e8e8e8",
  "TEXTURE": "./textures/WALL_01.jpg",
  "REPEAT": [2, 2],
  "METALNESS": 0.0,
  "ROUGHNESS": 0.95
}
```

- `MATERIAL_NAME` має збігатися з назвою матеріалу в Blender/GLB (паспорт: `MAT_WALL_01`, `MAT_FLOOR`, тощо).
- `TEXTURE` — шлях до файла текстури у вашому проєкті.

Найпростіші варіанти:

Ідея однакова: ви завантажуєте папку проєкту, а сервіс публікує її як сайт. Глядачу нічого встановлювати не треба — тільки відкриває посилання у браузері.
